--------------------------------------------------------------------------------
  DESIGN PHILOSOPHY
--------------------------------------------------------------------------------

We want a COMPLETELY NEW, MODERN, PREMIUM design for this admin form page.
Do NOT copy or mimic any existing LMS tools (Moodle, Canvas, Udemy admin).
Think: "What would a world-class SaaS product like Notion, Linear, or Figma
design for an admin category creation form in 2025?"

The design should feel:
  ✦ Visually rich — not a plain white form, but an experience
  ✦ Graphic-forward — use illustrations, gradient blobs, glassmorphism,
    abstract shapes, subtle patterns, or 3D-style visual elements
  ✦ Premium and editorial — every pixel should feel intentional
  ✦ Fun and motivating — creating a category should feel exciting, not boring
  ✦ Both Light & Dark modes must feel premium (NOT just color-swapped)

The color palette will be provided separately by the design team.
The AI should pick an ideal layout, card styles, typography, spacing, and
visual hierarchy completely from scratch.

--------------------------------------------------------------------------------
  PURPOSE OF THE SCREEN
--------------------------------------------------------------------------------

This is an admin-facing form page inside an LMS (Learning Management System).
An admin user visits this page to create a new "Category" — which is a top-level
container for organizing courses. Think of it like creating a "department" or
a "subject area" (e.g., "Data Science", "Cloud Computing", "Frontend Dev").

Categories appear as visual cards on the student-facing portal with a colored
header, icon/image, and description. So what the admin designs here directly
affects the public visual output.

This means the page should feel like a "design studio" or "brand kit builder" —
the admin is crafting a visual identity for a category, not just filling a form.

--------------------------------------------------------------------------------
  FEATURES TO DESIGN (must all be included)
--------------------------------------------------------------------------------

FEATURE 1: name (Category Name)
  - A prominent, large text input — this should be the hero element of the form
  - As the user types, a "live slug preview" should appear below it,
    showing what the auto-generated URL looks like (e.g. /categories/data-science)
  - Should feel more like typing a "title" than filling a form field
  - Validation: required field. Must show meaningful empty/error state.

FEATURE 2: iconType, icon, & thumbnailImageUrl (Icon / Thumbnail Picker)
  - The user can choose between two modes (iconType):
      MODE A — Emoji: A character-based emoji as the category icon (stored in 'icon').
        Design should include a visual emoji preview (large, prominent),
        a text field to type/paste an emoji, and an interactive emoji picker
        panel that opens on demand (floating popup with emoji categories + search)
      MODE B — Image: A photo or illustration as the category thumbnail (stored in 'thumbnailImageUrl').
        Design should include a drag-and-drop upload zone with a visual drop
        animation, a file picker button, and an alternative "paste URL" input.
        Once uploaded, show a rich image preview with a hover-to-delete overlay.
  - The toggle between modes should be visually engaging — pill buttons, icon tabs,
    or a toggle-style switcher. Make it feel like switching between modes is fun.

FEATURE 3: description
  - A large, generous textarea for the category description
  - KEY FEATURE: An "Generate with AI" magic button right next to the label
      → This button sends the category name to an AI and auto-fills the description
      → Must have THREE visual states: default, loading (animated spinner), success
      → Loading state should be visually interesting (shimmer, pulse, or animated wand icon)
  - Hint text: "Appears in course listings and SEO previews."

FEATURE 4: color (Accent Color Selector)
  - This is a KEY visual feature — the color chosen is used as the category's
    brand color across the entire platform (card borders, badges, highlights)
  - Design should include:
      → A visual color wheel / custom color picker
      → A row of 6 preset color swatches (circles) for quick selection
      → A HEX value display showing the current color code
  - Make this section feel like a mini "brand color studio" element
  - The selected color should dynamically update the live preview card

FEATURE 5: isActive (Status Toggle)
  - A simple Active / Inactive toggle switch
  - Active = category is visible to learners
  - Inactive = hidden, like a draft state
  - Design should visually communicate the difference between these two states
    in a clear, graphic way — not just a colored pill but something that feels alive.
    Maybe an animated state change, a visual metaphor (eye open/closed), or a
    status card that changes completely.

FEATURE 6: LIVE CARD PREVIEW (CRITICAL FEATURE)
  - As the admin fills in the form (name, icon, color, description, status),
    a LIVE PREVIEW updates in real time showing exactly how the category card
    will look on the student portal
  - This should be a premium design moment — perhaps a floating "preview panel"
    or an animated card that morphs as inputs change
  - Include: category name, auto-generated slug, description preview (2-3 line clamp),
    icon/thumbnail, accent color, status badge (Active/Inactive), and
    two stats: "0 Courses" and "0 Learners"
  - Make this feel like a product mockup or a live brand preview in a design tool

FEATURE 7: FIELD COMPLETION TRACKER
  - A visual checklist or progress tracker showing which required fields are filled
  - Fields tracked: name, icon, description, color, isActive
  - Should use checkmarks (✓ / ○) with visual state changes as fields are completed
  - Could be a progress bar, step tracker, or animated checklist

FEATURE 8: NAVIGATION / BREADCRUMB
  - Admin comes from: Dashboard → Categories → Create
  - Breadcrumb should be clean, minimal, and modern

FEATURE 9: QUICK TIPS / GUIDANCE
  - A small helper section with 4 tips for creating good categories
  - Should feel like a "pro tips" card — use icons, good typography, maybe a
    subtle background gradient or frosted glass style

FEATURE 10: ACTION BUTTONS (bottom of the page / footer bar)
  - CANCEL button — navigates back without saving
  - SAVE AS DRAFT button — saves with inactive status (won't be visible to learners)
  - CREATE CATEGORY button — primary action, full save and publish
  - Also show an "Auto-saved · just now" indicator somewhere to reassure the user

--------------------------------------------------------------------------------
  MODALS / OVERLAYS TO DESIGN
--------------------------------------------------------------------------------

Design beautiful versions of these modals (they appear on errors):

  MODAL A — Duplicate Category Error:
    User tried to create a category with a name that already exists.
    Content: error icon, "Duplicate Found" title, explanation text, dismiss button.

  MODAL B — Validation / Missing Fields Error:
    User tried to submit without filling required fields.
    Content: alert icon, "Missing Fields" title, message, dismiss button.

Both modals should have a frosted-glass or premium overlay backdrop.

--------------------------------------------------------------------------------
  VISUAL ELEMENTS TO INCORPORATE
--------------------------------------------------------------------------------

We ENCOURAGE the designer to add visual richness using:

  ✦ Subtle gradient background or mesh/noise texture on the page bg
  ✦ Glassmorphism cards (frosted/blurred card panels)
  ✦ Decorative blobs or abstract shapes in section backgrounds
  ✦ Smooth animated transitions when switching between states
  ✦ The emoji preview should feel like a product feature, not a form input
  ✦ The drag-and-drop zone should have a stylish hover animation
  ✦ Category color picker swatch selection should have a satisfying visual click
  ✦ The live preview card should have a subtle floating shadow and glow effect
    in the category's chosen color
  ✦ Section headers with small decorative icons or accent bars
  ✦ Consider a side-panel design, a floating modal-like form, a wizard approach,
    or a multi-panel studio layout — pick what feels best and most modern

--------------------------------------------------------------------------------
  BOTH LIGHT & DARK MODE REQUIRED
--------------------------------------------------------------------------------

Please provide TWO complete screen designs:

  LIGHT MODE:
    - Should feel clean, airy, and professional
    - Could use a very subtle warm or cool off-white background
    - Cards should feel crisp and elevated (not flat white blocks)
    - Color accents should pop against the light background

  DARK MODE:
    - Should feel deeply rich, premium, and not just "gray everywhere"
    - Use layered dark tones (very dark bg, slightly lighter cards)
    - Neon or vibrant color accents should glow subtly in dark mode
    - The live preview card especially should feel dramatic in dark mode

  ✗ Do NOT make a generic flat white form with gray borders
  ✗ Do NOT make it look like Google Forms or a standard CRUD admin
  ✗ Do NOT use generic sans-serif at default sizes with default spacing
  ✗ Do NOT just swap colors between light and dark mode
  ✗ Do NOT skip the live preview — it is a HERO feature
  ✗ Do NOT use overly complex navigation — this is a focused creation task
